# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a personal Emacs configuration (~/.emacs.d) for web/TypeScript development. It uses a modular structure with `use-package` for package management.

## File Structure

- `init.el` — Entry point; loads other files, configures packages via `use-package`, sets global keybindings and misc settings
- `functions.el` — All custom interactive functions and hooks (loaded early by `init.el`)
- `fish-colors.el` — Syncs Emacs theme colors to fish shell when running inside Emacs
- `helm-magit-recent-branches.el` — Custom Helm source for switching git branches (sorted by recency with commit messages)
- `custom.el` — Auto-generated by `M-x customize`; loaded last by `init.el`. Holds theme, package list, face customizations, and editor defaults

## Key Architectural Decisions

**Two-space indentation** for JS/TS/Web is set in `custom.el` (not `init.el`).

**TypeScript setup**: Uses `typescript-ts-mode` (tree-sitter based) + `tide` for IDE features. The `setup-tide-mode` function in `functions.el` is the hook that activates tide, flycheck, eldoc, and company together. Note: `tide-setup` is NOT in the `:hook` of `use-package tide` — it's in the `:init` of `use-package typescript-ts-mode`.

**Magit advice**: Three `advice-add` calls patch magit behavior in `init.el`:
- Forces branch upstream to always be `origin` (overrides `magit-branch-maybe-adjust-upstream`)
- Fixes diff range from `...` to `..` (filters `magit-diff--dwim` return value)
- Strips tslint warnings from JSON before flycheck parses eslint output

**Mode line**: The branch name in the mode line is shortened via an advice on `vc-git-mode-line-string` to show only the `PORT-NNNN` ticket number.

**Perspective mode**: Used for workspace management. `C-c M-p` prefix. A performance patch overrides `persp-current-buffers*` to avoid slowness with the global perspective.

**Copilot**: Manually triggered only (no auto-mode hook). Bound to `C-c c` prefix.

## Reload Workflow

To apply changes to `init.el` or `functions.el` without restarting Emacs:
- `M-x eval-buffer` when the file is current buffer
- Or `M-x load-file RET ~/.emacs.d/init.el`

To reload a single `defun`: place point inside it and `M-x eval-defun` (or `C-M-x`).

## Common Keybindings (Custom)

| Key | Function |
|-----|----------|
| `C-a` / `<Home>` | Toggle between indentation and line start |
| `C-o` | Insert blank line above, with indentation |
| `M-;` | Comment/uncomment line |
| `C-M-;` | Copy line and comment the original |
| `C-x g` | `magit-status` |
| `C-c b` | Helm recent branches |
| `C-c p f` | `helm-ls-git` (file search in git repo) |
| `C-c p s` | `helm-projectile-rg` (ripgrep search) |
| `C-c a c/t/s/j` | Angular: open component/template/stylesheet/spec |
| `M-SPC` | Company complete |
| `F12` | YASnippet expand |

## Angular Navigation

`C-c a` shortcuts (`my-angular-open-*` in `functions.el`) navigate between files of an Angular component. They parse the current filename with a regexp expecting the pattern `name.component.ext` or `name.service.ext`. HTML/SCSS shortcuts are disabled for service files.

## Snippets

YASnippet templates live in `snippets/typescript-mode/` and `snippets/typescript-ts-mode/`. Tab key is unbound from yasnippet — use `F12` to expand.
